# 漢直ストリーム統計を実装

漢直で連続して入力している状態を「ストリーム」と定義し、ストリーム長さに対する頻度を集計する

* 「連続していない」の定義
    * 前回の入力文字確定から次の確定までの時間が一定時間経過
        * しきい値として0.5秒、1秒で2種類のデータセットを取る
        * 設定項目や統計情報のキーとして文字列 "0.5", "1.0"を使い、計測にはこれを数値化したものを使う
    * PendingKakuteiの受容は入力文字確定とカウントし、前回の確定からの時間が短かければストリームは連続と見なす
    * モード切りかえで全角モード、一行入力モードにした
    * MacTcodeをオフにした
    * Backspaceを押した(ただし、backspaceIgnoreで無視されているものは除く)
* 漢直ストリームの長さ統計
    * ストリーム長さの計算方法
        * 連続している確定の文字数をカウントする
        * 部首変換の確定、交ぜ書き変換の確定では、ヨミ分はすでにストリーム長さとしてカウント済みなので、その分を減算する
            * 交ぜ書き変換の確定では、置換される文字数分のみカウントする。ヨミ文字列として取得した文字数と混同しないこと
    * ストリームの長さが決まったとき(不連続が検出されたとき)に、長さをキーとしたヒストグラムを更新する
        * ヒストグラムは要素数51の配列でカウントする
        * 長さが50を超えるときは50でキャップする
    * writeStrokeStatsToFileのタイミングでファイルに記録する
        * この時点でまだ連続しているストリームの情報は反映されなくてよい
* 統計データの蓄積
    * stroke-stats.jsonにstreamCountを追加し、統計情報はセッションをまたいで永続化する
    * キーは連続判定のしきい値を文字列で表現したもの
```
"streamCount" : {
  "0.5" : [0, 5, 5, ... ],
  "1.0" : [0, 7, 9, ... ]
}
```

* ストリーム統計に関する設定項目
    * streamStatsEnabledを追加
    * streamThresholds: ["0.5", "1.0"]
    * config.json内に以下を追加(いずれもデフォルト値はtrue)
        * "streamStatsEnabled"
        * "strokeStatsEnabled" (以前の機能実装時の追加忘れ)
    * config.jsonの読み込み、sample-config.json、ConfigParams.mdを更新する
